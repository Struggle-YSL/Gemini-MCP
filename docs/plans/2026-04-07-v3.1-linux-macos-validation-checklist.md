# v3.1 Linux/macOS 平台支持验收清单（进行中）

> 文档日期：2026-04-07
> 适用版本：v3.1（进行中）

## 1. 目标

在保留 Windows 兼容能力的前提下，补齐 Linux/macOS 的安装发现与代理兜底，形成可复用的跨平台运行基线。

## 2. 已完成（首批落地）

- 新增 `src/gemini-runner-discovery.ts`（跨平台路径探测）
  - `GEMINI_PATH` 显式覆盖
  - PATH / `which` / `where` 探测
  - npm / pnpm / yarn 全局 bin 探测
  - 平台常见目录兜底（含 `/opt/homebrew/bin`、`~/.local/bin` 等）
- `GeminiResolution` 增强：新增 `searchedPaths`
- `runtime-diagnostics` 增强：新增
  - `gemini_runtime.searched_paths`
  - proxy source 扩展（`macos-scutil` / `linux-gsettings`）
- Unix 代理兜底策略补齐（`src/gemini-runner-proxy.ts`）
  - 环境变量 `HTTP_PROXY` / `HTTPS_PROXY` / `ALL_PROXY`
  - macOS `scutil --proxy`
  - Linux GNOME `gsettings`
- 新增单测
  - `test/gemini-runner-discovery.test.mjs`
  - `test/gemini-runner-proxy.test.mjs`
- 新增跨平台三场景安装冒烟脚本
  - `scripts/check-v31-platform-smoke.mjs`
  - npm script：`npm run check:v31-smoke`（默认保留报告；可加 `-- --cleanup-temp` 自动清理临时目录）
- 新增 CI 跨平台冒烟矩阵
  - `.github/workflows/ci.yml` 新增 `v31-platform-smoke` job（ubuntu/macos/windows）
  - 每个平台执行 `npm run check:v31-smoke` 并上传报告 artifact（避免报告文件被清理）

## 3. 本地验证（2026-04-07）

已执行：

```bash
npm run build
node --test test/gemini-runner-discovery.test.mjs test/gemini-runner-proxy.test.mjs test/gemini-runner.test.mjs
npm run check:v31-smoke
```

结果：通过（当前平台 win32）。
