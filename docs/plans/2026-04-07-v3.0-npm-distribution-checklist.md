## 4. Windows 三场景安装验收（2026-04-07，本地模拟 clean machine）

> 说明：当前为“本地 tarball + 隔离目录”模拟验证，用于快速确认发布工件可安装可运行；真实 clean machine（全新系统）验收仍需补录。

前置：

```bash
npm pack
# 产物：gemini-mcp-1.0.0.tgz
```

### 场景 A：npx 零安装（本地 tarball 模拟）

```bash
npx --yes --package .\gemini-mcp-1.0.0.tgz gemini-mcp --version
npx --yes --package .\gemini-mcp-1.0.0.tgz gemini-mcp --help
```

结果：通过（输出版本 `1.0.0`，help 正常）。

### 场景 B：全局安装（`--prefix` 隔离模拟）

```bash
npm install -g .\gemini-mcp-1.0.0.tgz --prefix D:\gemini-mcp\test-tmp\global-prefix
D:\gemini-mcp\test-tmp\global-prefix\gemini-mcp.cmd --version
D:\gemini-mcp\test-tmp\global-prefix\gemini-mcp.cmd --help
```

结果：通过（输出版本 `1.0.0`，help 正常）。

### 场景 C：项目依赖安装（隔离项目模拟）

```bash
# 在 D:\gemini-mcp\test-tmp\project-dependency
npm init -y
npm install -D D:\gemini-mcp\gemini-mcp-1.0.0.tgz
node .\node_modules\gemini-mcp\dist\cli.js --version
node .\node_modules\gemini-mcp\dist\cli.js --help
```

结果：通过（输出版本 `1.0.0`，help 正常）。
